Feijiang Han, University of Pennsylvania

From Passive Detection to Proactive Defense

For years, cybersecurity has focused on detecting Webshells—a critical first step. But just knowing a server is compromised isn't enough. To mount a truly effective defense, we need to understand the attacker's identity and intent.

The Limits of Binary Detection

Standard detection simply answers "yes" or "no" to whether a file is malicious. This approach offers limited intelligence for security operations, leaving teams in the dark about the nature of the threat. It's like a smoke alarm that tells you there's a fire, but not where it is or how it started.

The Need for Family Classification

This research moves beyond detection to family classification: identifying a Webshell's specific lineage. Automating this task unlocks deep threat intelligence, helping security teams attribute attacks, anticipate an adversary's next moves, and deploy a faster, more targeted response before significant damage occurs.

A Framework for Understanding Malware Behavior

To classify Webshell families, we developed a systematic framework that captures their core behavior. This process turns raw, obfuscated code into structured, meaningful representations that machine learning models can analyze.

1

Capture Dynamic Behavior

We execute suspicious files in a secure sandbox to record their dynamic function call traces. This captures the Webshell's actual runtime behavior, bypassing common evasion techniques like encryption and obfuscation.

2

Abstract the Data

The raw traces are abstracted into three fundamental data structures, each providing a unique view of the program's logic:

  • Sequences: A linear, chronological list of function calls.
  • Graphs: A network showing all "who-calls-whom" relationships.
  • Trees: A hierarchical structure preserving the exact execution path and context.
3

Learn & Benchmark

We use a comprehensive suite of representation methods—from classic embeddings (Word2Vec) to Transformers (BERT) and Graph Neural Networks (GNNs)—to convert these structures into numerical vectors. We then benchmark their quality using standard classifiers.

Key Insights and Findings

Our extensive experiments on four real-world datasets revealed a clear hierarchy of effectiveness. The results provide a foundational baseline for future research and practical application.

Insight 1: Structure Definitively Outperforms Syntax

The most significant finding is that structural representations (graphs and trees) decisively outperform sequential ones. While sequence models like BERT are powerful, they treat code like a sentence, focusing on linear order.

However, a Webshell family's true signature lies in its architectural blueprint—the complex, non-local relationships between functions. Structural models capture this topology, making them far more robust against code shuffling and obfuscation.

Insight 2: Hierarchical Context is Crucial, Giving Trees an Edge

While both graphs and trees were effective, tree-based models yielded the best overall performance. A graph provides an aggregate view, but a tree preserves the precise execution path and call stack.

This hierarchical context is vital. For example, a malicious function like `eval()` may be used in different ways. A tree disambiguates these uses by showing their unique parent-child relationships, providing a richer, more discriminative fingerprint for classification.

Insight 3: GNNs are the Premier Architecture for Learning Behavior

Among all models, Graph Neural Networks (GNNs) emerged as the most powerful and stable architecture. GNNs are purpose-built to learn from relational data. Unlike classic methods that count predefined patterns, GNNs automatically learn the most important structural motifs that define a family's behavior.

The Graph Attention Network (GAT) was particularly effective, as it learns to focus on the most diagnostically critical function calls within the program's structure.

Practical Guidance for Implementation

Our findings translate directly into actionable guidance for security teams and researchers building next-generation defense systems.

Best Overall Performance

For the most consistent, top-tier results across both supervised (labeled data) and unsupervised (threat discovery) tasks, a Tree-GAT model is the recommended choice.

For Supervised Classification

When high-quality labeled data is available, GNNs are superior. GAT and GCN models paired with Random Forest or SVM classifiers deliver high-precision results.

For Unsupervised Threat Discovery

To discover new or unknown families, structural representations are critical. A Tree Kernel or Tree-GAT model can effectively group new malware samples, flagging emergent clusters for expert analysis.

Conclusion and Future Work

This study establishes the first systematic benchmark for fine-grained Webshell family classification, demonstrating that the key to understanding malware lies in its structural behavior, not just its code. By proving the superiority of tree-based representations and GNN architectures, we provide a foundational roadmap for the security community to build more intelligent, proactive, and resilient defense systems.

Future work will focus on advancing these capabilities through continual learning to adapt to evolving threats and enhancing model interpretability to provide security analysts with even deeper insights.
